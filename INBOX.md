# StrictDoc website

## StrictDoc project

- Software for technical documentation and requirements management.
- Open source requirements tool

## Main points

1) Bridging requirements and software.
  - Making requirements first-class citizen in software development.
  - Developers work effectively with requirements during software development.

2) Traceability between documents/requirements/models and source code, tests, test reports

3) Two workflows: CLI/IDE and Web.

4) API-first. Everything is scriptable and configurable. Adapting to project specifics.

5) Development with AI based on requirements.
- AI likes to work with requirements.

6) Documentation and requirements are stored in plain text, in Git. There is no database that hides from eye or control.

7A) Text formats that enable traceable documentation (content + meta (показать визуалом)). Strict-doc + Markdown + ReqIF.
7B) Formats: SDoc, Markdown, RST, ReqIF, JSON, PDF, Excel.

8) Used by both Agile projects and companies doing security- and safety-critical software developments (IEC 62443, DO-178C).

## Project

* Python. Pip package. Not Java, no Rust, no C++.
* Developed by critical software enthustiasts.
* Apache 2
* Created in 2019
