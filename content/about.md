# About StrictDoc

StrictDoc is a Python-based tool for technical documentation and requirements management.

- **Open source**, Apache 2.0 license
- **Started in 2019**
- **Developed in Python**

It was built by engineers developing critical software who wanted requirements management that fits naturally into a developer workflow: plain text, Git, and the command line. Over time, we expanded the tool with web-based document editing and many other features.

2) [Achieving end-to-end traceability in software projects using open source tools (2025)](https://github.com/strictdoc-project/strictdoc/blob/main/about/2025_ESA_SW_Product_Assurance_Workshop.pdf)

This is a short presentation about StrictDoc, delivered at the ESA Software Product Assurance Workshop 2025.

4) [Exploring possibilities for integrating StrictDoc and ELISA’s requirements template approach for the Linux kernel](https://lpc.events/event/19/contributions/2198/attachments/1884/4056/Linux%20Requirements%20with%20StrictDoc.pdf)

The talk argues that ELISA's Linux kernel safety documentation model and StrictDoc's requirements tooling are largely compatible and could be unified, with StrictDoc filling remaining gaps like drift detection and ELISA adopting it as a reference tool.

5) [Integrating StrictDoc with ELISA's Linux kernel requirements approach - ELISA Project Workshop 2025](https://www.youtube.com/watch?v=f5v6q3OFv88)

The talk introduces the StrictDoc project, demonstrates a traceability use case at Linutronix, and provides a hands-on showcase of how StrictDoc could support structured, maintainable requirements workflows for the Linux kernel. The session explores how requirements engineering practices can be enhanced within the kernel ecosystem, with an emphasis on improving traceability, documentation quality, and alignment with ELISA's requirements template approach.

6) [Application of the SPDX Safety Profile in the Safety Scope of the Zephyr Project (2024)](https://ftp.fau.de/fosdem/2024/k4401/fosdem-2024-3211-application-of-the-spdx-safety-profile-in-the-safety-scope-of-the-zephyr-project.mp4)

This presentation introduces the SPDX Safety Profile and details its application within the context of the Zephyr Project. There is also a presentation of the Zephyr project's methodology for capturing requirements using StrictDoc and a discussion of the upcoming plans for the integration of SPDX into StrictDoc.
